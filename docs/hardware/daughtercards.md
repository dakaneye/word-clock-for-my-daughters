# Daughtercard Mechanical Measurements

**Status:** v1 — measurements complete, custom footprints generated at
`hardware/word-clock.pretty/`. Combines the 1×N pin socket/header with
NPTH mounting holes locked relative to pin 1.

All three daughtercards plug into the main PCB on **B.Cu (back side)**. The
"top" orientation in the tables below is the user's caliper-measurement
orientation: pin row up, looking at the *component* side of the daughtercard.

## Custom footprints

Generated and checked into `hardware/word-clock.pretty/`:

- `Daughtercard_SD_HW125.kicad_mod` — 1×6 horizontal pin socket + 4× ⌀2.0 mm NPTH at corners
- `Daughtercard_RTC_DS3231.kicad_mod` — 1×6 horizontal pin socket + 3× ⌀3.0 mm NPTH
- `Daughtercard_AMP_MAX98357A.kicad_mod` — 1×7 vertical pin header + 2× ⌀2.6 mm NPTH at far short edge

The library is registered as `word-clock` in `hardware/fp-lib-table`. To
swap a daughtercard's footprint in eeschema, edit the symbol's Footprint
property to e.g. `word-clock:Daughtercard_SD_HW125`, then push the change
to PCB via Tools → Update PCB from Schematic. The new footprint replaces
the old socket and the mounting holes appear locked to pin 1 — drag the
connector and the holes follow.

Generated by `/tmp/gen_daughtercard_footprints.py` (which can be re-run if
measurements change).
